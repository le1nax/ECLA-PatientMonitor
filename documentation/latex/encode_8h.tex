Compresses input stream to output stream. 

The values {\ttfamily \texorpdfstring{$\ast$}{*}available\+\_\+in} and {\ttfamily \texorpdfstring{$\ast$}{*}available\+\_\+out} must specify the number of bytes addressable at {\ttfamily \texorpdfstring{$\ast$}{*}next\+\_\+in} and {\ttfamily \texorpdfstring{$\ast$}{*}next\+\_\+out} respectively. When {\ttfamily \texorpdfstring{$\ast$}{*}available\+\_\+out} is {\ttfamily 0}, {\ttfamily next\+\_\+out} is allowed to be {\ttfamily NULL}.

After each call, {\ttfamily \texorpdfstring{$\ast$}{*}available\+\_\+in} will be decremented by the amount of input bytes consumed, and the {\ttfamily \texorpdfstring{$\ast$}{*}next\+\_\+in} pointer will be incremented by that amount. Similarly, {\ttfamily \texorpdfstring{$\ast$}{*}available\+\_\+out} will be decremented by the amount of output bytes written, and the {\ttfamily \texorpdfstring{$\ast$}{*}next\+\_\+out} pointer will be incremented by that amount.

{\ttfamily total\+\_\+out}, if it is not a null-\/pointer, will be set to the number of bytes compressed since the last {\ttfamily state} initialization.

Internally workflow consists of 3 tasks\+:
\begin{DoxyEnumerate}
\item (optionally) copy input data to internal buffer
\item actually compress data and (optionally) store it to internal buffer
\item (optionally) copy compressed bytes from internal buffer to output stream
\end{DoxyEnumerate}

Whenever all 3 tasks can\textquotesingle{}t move forward anymore, or error occurs, this method returns the control flow to caller.

{\ttfamily op} is used to perform flush, finish the stream, or inject metadata block. See \doxylink{encode_8h_aed29c96cccbe4dc2aed53a2f1e2d04cb}{Brotli\+Encoder\+Operation} for more information.

Flushing the stream means forcing encoding of all input passed to encoder and completing the current output block, so it could be fully decoded by stream decoder. To perform flush set {\ttfamily op} to \doxylink{encode_8h_aed29c96cccbe4dc2aed53a2f1e2d04cba35d9c34839f1171ecb715de795e47664}{BROTLI\+\_\+\+OPERATION\+\_\+\+FLUSH}. Under some circumstances (e.\+g. lack of output stream capacity) this operation would require several calls to \doxylink{encode_8h_ac94b211d362a1a3d4f208a5736cc41a5}{Brotli\+Encoder\+Compress\+Stream}. The method must be called again until both input stream is depleted and encoder has no more output (see \doxylink{encode_8h_a122ad712b0987e513af6ce9ad90fc65f}{Brotli\+Encoder\+Has\+More\+Output}) after the method is called.

Finishing the stream means encoding of all input passed to encoder and adding specific \"{}final\"{} marks, so stream decoder could determine that stream is complete. To perform finish set {\ttfamily op} to \doxylink{encode_8h_aed29c96cccbe4dc2aed53a2f1e2d04cba9e7ec104794a42c7a14e3bec4753be18}{BROTLI\+\_\+\+OPERATION\+\_\+\+FINISH}. Under some circumstances (e.\+g. lack of output stream capacity) this operation would require several calls to \doxylink{encode_8h_ac94b211d362a1a3d4f208a5736cc41a5}{Brotli\+Encoder\+Compress\+Stream}. The method must be called again until both input stream is depleted and encoder has no more output (see \doxylink{encode_8h_a122ad712b0987e513af6ce9ad90fc65f}{Brotli\+Encoder\+Has\+More\+Output}) after the method is called.

\begin{DoxyWarning}{Warning}
When flushing and finishing, {\ttfamily op} should not change until operation is complete; input stream should not be swapped, reduced or extended as well.
\end{DoxyWarning}

Acquires pointer to internal output buffer. 

This method is used to make language bindings easier and more efficient\+:
\begin{DoxyEnumerate}
\item push data to \doxylink{encode_8h_ac94b211d362a1a3d4f208a5736cc41a5}{Brotli\+Encoder\+Compress\+Stream}, until \doxylink{encode_8h_a122ad712b0987e513af6ce9ad90fc65f}{Brotli\+Encoder\+Has\+More\+Output} returns BROTLI\+\_\+\+TRUE
\item use \doxylink{encode_8h_a2bda817fc6d2bbd9740115590ea93aa5}{Brotli\+Encoder\+Take\+Output} to peek bytes and copy to language-\/specific entity
\end{DoxyEnumerate}

Also this could be useful if there is an output stream that is able to consume all the provided data (e.\+g. when data is saved to file system).

\begin{DoxyAttention}{Attention}
After every call to \doxylink{encode_8h_a2bda817fc6d2bbd9740115590ea93aa5}{Brotli\+Encoder\+Take\+Output} {\ttfamily \texorpdfstring{$\ast$}{*}size} bytes of output are considered consumed for all consecutive calls to the instance methods; returned pointer becomes invalidated as well.
\end{DoxyAttention}
\begin{DoxyNote}{Note}
Encoder output is not guaranteed to be contiguous. This means that after the size-\/unrestricted call to \doxylink{encode_8h_a2bda817fc6d2bbd9740115590ea93aa5}{Brotli\+Encoder\+Take\+Output}, immediate next call to \doxylink{encode_8h_a2bda817fc6d2bbd9740115590ea93aa5}{Brotli\+Encoder\+Take\+Output} may return more data.
\end{DoxyNote}
